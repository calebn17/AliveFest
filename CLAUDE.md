## Project Overview

ALIVE Festival website — a static site for a one-day fitness and wellness festival (September 19, 2026) in honor of Wendy Tam, at Washington Park, Burlingame, CA. Originally designed in Framer and exported as static HTML via HTTrack Website Copier.

## Architecture

This is a **no-build static site**. There is no bundler, framework, or build step. All pages are self-contained HTML files with inline CSS/JS generated by Framer's export.

- `allivefest.website/` — the deployable site root (this is what GitHub Pages serves)
  - `index.html` — homepage
  - `day.html` — day-of schedule overview
  - `partners.html` — partners listing
  - `day/` — subpages: `classes.html`, `vendors.html`, `rare-cancer-research.html`
  - `partners/` — subpages: `mucinous-ovarian-cancer-coalition.html`, `next-play.html`
  - `image_assets/` — local copies of hero/OG images used by the site
- `image_assets/` — source copies of the same images (at repo root, outside the deploy folder)

## Deployment

Pushes to `main` auto-deploy via GitHub Actions (`.github/workflows/deploy.yml`). The workflow uploads `allivefest.website/` as a GitHub Pages artifact — no build step involved.

## Local Development

Open any HTML file directly in a browser, or serve locally:

```
python3 -m http.server 8000 -d allivefest.website
```

## Key Conventions

- **Framer export origin**: The HTML is machine-generated. Each file contains a large inline `<style>` block and a single `<script>` tag with the full Framer runtime and page content as minified JS. Manual edits should be targeted (e.g., swapping image URLs, updating meta tags) rather than restructuring the generated markup.
- **Image hosting migration**: OG/Twitter meta images have been moved from `framerusercontent.com` CDN URLs to local `image_assets/` paths. When adding new pages or updating meta tags, use relative paths to `image_assets/` (e.g., `../image_assets/Photo%20Name.png` from subpages).
- **File naming**: Image filenames contain spaces (e.g., `Girl at Fitness Festival.png`). Use `%20` encoding in HTML references.
- **No package.json / no dependencies**: There are no npm packages, linters, or test suites to run.
